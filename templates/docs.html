RoosterAPI

RoosterAPI is een A.P.I. (application programming interface) voor het rooster van het horizon college.
Deze A.P.I. Kan worden gebruikt om een GET request te sturen voor een lijst met objecten, waar de informatie van de vakken in staan.

De A.P.I. wordt gehost met heroku, op de volgende url: https://horizon-rooster.herokuapp.com/api/rooster/

Output

De output van de A.P.I. is een lijst met JSON objecten. Deze objecten hebben allemaal dezelfde opbouw, maar andere informatie. Hieronder is een voorbeeld


	{
		"vak": "ENG",
		"docent": "WEELD0",
		"plaats": "online",
		"wanneer": {
			"dag": "Dinsdag 13-04",
			"tijd": {
				"start": "09:20",
				"eind": "10:10"
			}
		}
	}
*Het 'plaats' kan ook het lokaalnummer bevatten.

Input

De url heeft een aantal variaties waarmee kan worden gespecificeerd welke lessen worden opgehaald.

De volgende urls worden aangenomen:

  • https://horizon-rooster.herokuapp.com/api/rooster/
  • https://horizon-rooster.herokuapp.com/api/rooster/today
  • https://horizon-rooster.herokuapp.com/api/rooster/tomorrow

De /today en /tomorrow geven alleen de lessen weer die vandaag of morgen te zien zijn.
Echter zijn er meer opties. Via URL Queries kan er meer worden gespecifeerd. Deze queries kunnen worden verdeelt in 2 soorten: Filters(laten je bijvoorbeeld een specifiek vak kiezen) en Url kiezers(helpend om te bepalen welk rooster wordt gebruikt).

De url queries kunnen natuurlijk ook worden gecombineerd, bijvoorbeeld ?week_nummer=16&vak=PORTF

Url Kiezers

  • ?sector=ECO

    Je kan specificeren uit welke sector het rooster wordt opgehaald. De standaard waarde is ECO

  • ?gebouw=HRN

    Je kan specificeren uit welk gebouw het rooster wordt opgehaald. De standaard waarde is HRN(hoorn)

  • ?klas=H19AO-A

    Je kan specificeren uit welke klas het rooster wordt opgehaald. De standaard waarde is H19AO-A. Om te bepalen wat je hier moet invullen, kijk je op de originele site, en vul je in wat er in het 'Element' vak staat.

  • ?week_nummer=15

    Je kan specificeren van welke week het rooster wordt opgehaald. De standaard waarde is de huidige week.
    Dit kan niet oneindig ver terug of vooruit. Meestal kan er 1 week terug en 2 weken vooruit gegaan worden, maar dit staat niet vast.

Filters

De filters veranderen niet wat voor data wordt opgehaald, alleen welke data wordt teruggegeven. Dit kan niet worden gebruikt bij /tomorrow en /today
(tenzij daar vraag naar is, in dat geval neem gerust contact op)

  • ?vak=PORTF

    filtert het vak. alleen de lessen van dit vak worden laten zien

  • ?docent=KERKCK0

    filtert de docent. alleen de lessen van deze docent worden laten zien

  • ?plaats=C207

    filtert het lokaal. alleen de lessen op de plaats worden laten zien

  • ?dag_van_week=2

    filtert de dag van de week. alleen de lessen op deze dag worden laten zien. Dit is zero-based.(maandag=0, dinsdag=1 etc.)
